# PeakRun AI 隐私政策

**生效日期：2026 年 7 月 15 日**

**个人信息处理者：熊翼（个人）**

**联系邮箱：danhabit@163.com**

PeakRun AI（以下简称"本应用"或"我们"）重视您的个人信息和隐私。本政策说明我们在您使用跑步记录、Apple 健康同步、训练计划、AI 教练、语音教练及相关功能时，如何收集、使用、存储、同步、共享和保护信息，以及您如何行使权利。

请在使用前认真阅读。健康、定位、伤病、语音等信息可能属于敏感个人信息；我们仅会在您主动启用相关功能、授予系统权限或另行确认后，基于明确目的处理。您可拒绝非必要权限，但对应功能可能无法使用。

---

## 1. 本政策适用范围

本政策适用于 PeakRun AI iPhone 应用、配套 Apple Watch 应用及应用内提供的相关功能，不适用于由第三方独立提供并适用其自身隐私政策的服务。第三方服务详见第 6 节。

## 2. 我们处理的信息

### 2.1 账号与个人资料

- **通过 Apple 登录信息**：Apple 用户标识，以及仅在 Apple 首次提供且您选择共享时取得的姓名和邮箱，用于登录、恢复登录状态和关联本地资料。
- **您填写的资料**：昵称、出生日期或年龄、性别、体重、跑步水平、训练目标、比赛日期、训练日偏好等，用于个性化展示、计算和训练计划。
- **训练与主观记录**：训练计划、完成情况、跑后感受、备注、教练记忆、对话、您主动记录的伤病部位和严重程度等，用于训练管理和个性化教练服务。

### 2.2 健康与运动信息

在您授权 Apple 健康（HealthKit）后，本应用可读取：

- 运动记录及其类型、时间、时长、距离、路线、活动能量、心率和跑步速度等；
- 心率、静息心率、步行平均心率、心率变异性（HRV）和最大摄氧量（VO₂ Max）；
- 步数、步行/跑步距离、骑行距离、游泳距离；
- 睡眠、呼吸频率、血氧饱和度、活动能量和基础能量。

本应用可将您在 PeakRun AI 中完成的跑步运动、距离和活动能量写入 Apple 健康。我们使用这些信息生成运动历史、心率区间、恢复和训练负荷等统计及 AI 教练上下文。**HealthKit 数据不会用于广告、营销、数据经纪或向第三方出售。**

### 2.3 定位、路线、城市与天气

- 跑步期间，在您授权后使用精确位置持续记录 GPS 点，以计算距离、配速、海拔和路线。
- 可通过 Apple 的定位与地理编码能力解析大致城市，用于实时语音教练的本地环境上下文。
- 为获取跑步时或计划日期的天气、湿度和空气质量，本应用会向 Open-Meteo 发送必要的经纬度、日期或时间。Open-Meteo 及网络服务商可能同时接收 IP 地址等常规网络信息。
- 因 iOS 后台跑步记录需要，本应用可在跑步已经开始后于后台继续使用定位。**我们不会为广告目的持续追踪您的位置。**

### 2.4 运动传感器

在您授权"运动与健身"后，本应用可在跑步期间使用步数、节奏及设备运动数据，用于估算步频、速度稳定性和跑步指标。

### 2.5 语音与音频

- 只有在您主动按住说话或启动相关语音功能时，本应用才会访问麦克风。
- 录制的语音片段可发送至您选择或本应用配置的语音识别服务（例如 OpenAI 兼容服务或火山引擎），以转换为文字。
- AI 教练回复文字可发送至所选语音合成服务生成音频，或由 Apple 系统语音在设备上朗读。
- **麦克风不会被用于广告监听。** 第三方语音服务对请求内容和日志的保留以其自身政策为准。

### 2.6 AI 教练与生成内容

当您发起 AI 对话、生成训练计划、跑后分析、信件或实时语音教练内容时，本应用可将完成任务所必要的以下信息发送到您选择或本应用配置的 AI 服务端点：

- 您输入的消息、语音识别文字及相关对话历史；
- 昵称、年龄、体重、训练目标、计划偏好和比赛日期；
- 近期运动摘要、距离、配速、时长、心率、心率区间、恢复、睡眠或其他健康摘要；
- 跑后感受与备注、教练记忆，以及您主动记录的伤病信息；
- 当前跑步的时间、距离、配速、心率、目标、城市和天气等实时摘要。

上述内容仅用于生成对应回复，**不用于广告**。请勿在对话中输入完成服务所不必要的身份证号、银行卡号、他人隐私等信息。

### 2.7 设置、运行与通知信息

- 本应用在设备上保存语言、提醒、训练、AI/语音服务端点等设置，并将 API 密钥保存在系统钥匙串。
- 可记录必要的本地诊断日志、同步游标和每日 API 调用次数，用于排障、避免重复同步、保障稳定性和控制服务用量。应用当前未集成第三方广告 SDK，也不会为跨应用追踪而收集数据。
- 在您授权通知后，本应用可发送训练、教练内容或信件提醒。您可在系统设置中关闭。

### 2.8 购买信息

如未来启用 App Store 订阅或购买，交易由 Apple 处理。我们可取得产品标识、订阅状态、有效期及交易校验结果以提供权益，但不会取得您的完整银行卡信息。

## 3. 信息使用目的

我们仅在以下必要范围内处理信息：

- 创建和维护账号、完成登录及账号删除；
- 记录和展示跑步、路线、训练历史及健康趋势；
- 计算配速、心率区间、训练负荷、恢复和计划完成度；
- 生成训练计划、AI 教练回复、跑后分析、信件和语音内容；
- 提供天气、空气质量、提醒、设备间同步和 Apple Watch 联动；
- 保障安全稳定运行、排查故障、恢复同步及防止滥用；
- 履行法律义务、响应合法请求及保护您、我们或他人的合法权益。

如需将信息用于本政策未说明且与原目的无直接关联的用途，我们会依法另行告知并在需要时取得您的同意。

## 4. 系统权限与您的选择

- **Apple 健康**：用于读取运动和健康指标，并写入您在本应用完成的跑步。可在 iOS"设置/健康"中随时调整。
- **定位**：用于跑步轨迹、配速、城市和天气。可在 iOS 设置中关闭；关闭后路线和依赖定位的功能不可用。
- **运动与健身**：用于步频和运动指标，可在 iOS 设置中关闭。
- **麦克风与语音**：用于按住说话和语音识别，可在 iOS 设置中关闭。
- **通知**：用于训练和教练提醒，可在 iOS 设置中关闭。
- **iCloud**：由您的 Apple 账号和系统设置控制；退出 iCloud 或关闭相应能力后，跨设备同步可能停止。

撤回授权不影响撤回前基于授权进行的处理。

## 5. 存储、同步与保留

1. **设备本地**：原始运动记录副本、HealthKit 同步的健康样本、睡眠记录、GPS 路线、运行缓存及部分设置主要存储在您的设备上。
2. **iCloud/CloudKit**：当设备已登录 iCloud 且同步可用时，个人资料、训练计划、AI 分析、信件、教练对话与记忆、伤病记录、计划变更记录、每日城市/天气摘要和 API 用量计数等可同步到您 Apple 账号对应的 iCloud 私有数据库。原始 HealthKit 数据库由 Apple 管理；本应用的本地健康样本和原始路线不放入本应用的 CloudKit 同步存储。
3. **系统钥匙串**：通过 Apple 登录凭据及您配置的 AI、语音识别和语音合成 API 密钥存储在系统钥匙串中。
4. **第三方处理**：发送至 AI、语音、天气或 Apple 服务的数据按完成请求所需时间及第三方政策保留。我们无法代替第三方承诺其具体保留期限。
5. **保留期限**：本应用数据通常保留至您删除相应内容、删除账号或不再需要实现本政策所述目的时；法律法规要求保留的除外。

您的个人信息在中国境内存储和处理，我们不会将您的个人信息传输至境外。

## 6. 共享、委托处理与第三方服务

**我们不出售、出租个人信息，不将 HealthKit 数据用于广告，也不与数据经纪商共享。** 仅在提供您主动使用的功能所必要时向以下服务发送数据：

- **Apple**：通过 Apple 登录、HealthKit、iCloud/CloudKit、定位/地理编码、通知、系统语音及 App Store 交易。Apple 按其隐私政策处理相关数据。
- **AI 模型服务**：默认或由您配置的 OpenAI 兼容端点等，用于处理第 2.6 节所述提示词和训练/健康上下文并生成内容。实际接收方取决于应用版本、运营配置或您填写的 Base URL。
- **语音服务**：OpenAI 兼容服务、火山引擎或您配置的兼容端点，用于语音识别或语音合成。
- **Open-Meteo**：接收必要的经纬度和时间以返回天气与空气质量。

不同服务提供方可能是独立个人信息处理者。使用自定义 API 地址表示您选择将相关请求发送给该地址的运营方，请在配置前核实其身份、安全性和隐私政策。

此外，我们可能在取得您的单独同意后共享信息，或根据法律法规、诉讼程序、行政司法机关依法提出的要求进行披露。发生合并、转让等主体变更时，我们会依法告知接收方并要求其继续受本政策约束。

## 7. 删除账号与您的权利

您可依法请求访问、更正、复制、删除个人信息，撤回同意，或注销账号：

- 在个人资料、历史、计划、信件和教练笔记等页面查看或修改信息；
- 在系统设置中撤回 HealthKit、定位、运动、麦克风、语音和通知权限；
- 在相应 AI/语音 API 设置中清除密钥和端点；
- 通过"个人资料 → App 设置 → 删除账号"删除本应用中的账号及业务数据。

删除账号会删除当前设备中的本应用模型数据，并对可同步的 CloudKit 记录发起删除；同步完成可能需要一定时间。该操作不会删除 Apple 健康中由 Apple 或其他应用保存的原始记录，也不能自动删除第三方 AI/语音服务已按其政策处理的数据。系统钥匙串中的自定义 API 密钥请先在对应设置中清除。

**仅卸载应用不等同于注销账号**，iCloud 记录或钥匙串项目可能仍然保留。您也可通过本政策顶部邮箱联系我们；为保护安全，我们可能需要核验您的身份。

## 8. 信息安全

我们采取与信息敏感程度相适应的措施，包括使用 iOS 沙盒、系统钥匙串、HealthKit 权限、iCloud 私有数据库和传输加密等。但互联网传输和电子存储不存在绝对安全。请保护好设备、Apple ID 和自定义 API 密钥，不要使用来源不明的服务端点。发生可能影响您权益的安全事件时，我们会依法采取补救并履行通知义务。

## 9. 未成年人

本应用不面向未满 14 周岁的儿童。我们不会在明知的情况下未经监护人同意处理儿童个人信息。如您是监护人并认为儿童向我们提供了信息，请联系我们，我们将依法核实并处理。

## 10. 政策更新

我们可因功能、第三方服务或法律要求变化更新本政策，并标明新的生效日期。发生重大变化时，我们会通过应用内提示、登录页面或其他合理方式通知；依法需要重新取得同意的，我们会另行征得同意。
